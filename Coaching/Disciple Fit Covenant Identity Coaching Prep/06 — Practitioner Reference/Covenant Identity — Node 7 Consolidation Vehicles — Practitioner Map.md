# Covenant Identity — Node 7 Consolidation Vehicles — Practitioner Map
*Practitioner Reference — the causal sequence of tools that carry receptive return, in the order the mechanism actually requires them*

*Related: [[Covenant Identity — Consolidation Through Receptive Return — Mechanism]] (why repeated receptive return, not repetition, consolidates — the mechanism this document sequences into vehicles); [[Covenant Identity — Assigning Receptive Return — Practitioner Technique]] (how to word the instruction for whichever vehicle is in use); [[Covenant Identity — Between-Session Library — Design Rationale]] (the full 15-tool library this document is not a replacement for — see Scope Boundary)*

---

**What this is:** A single-page map of the specific tools that carry Node 7 consolidation, ordered by when each one operates relative to a session where covenant truth landed — not by tool name, phase number, or library category. The Between-Session Library Design Rationale document inventories all 15 between-session tools; this document answers a narrower question: *of everything in that library, which pieces specifically carry receptive return, and in what causal order do they operate around a single Node 5-6 event?*

**When used:** Immediately after a session where covenant truth has landed (Node 5-6 complete), when deciding what to assign and in what sequence. When a client's consolidation has stalled and the practitioner needs to check which vehicle in the sequence is missing, not just whether "between-session work" in general is happening.

**Who uses it:** Practitioner only.

---

## Why Order Matters Here

The Consolidation mechanism document establishes that formation stabilizes through *repeated* receptive return, not a single reception event — but it doesn't specify which vehicle carries that return at which point in time relative to the session. Treating all between-session tools as interchangeable, assignable in any order, misses something the mechanism actually implies: the vehicle immediately after a session (while the encoding is still labile) is doing different work than the vehicle three days later, which is doing different work than the vehicle that only activates when the client is around another person. Sequence is not administrative — it follows the shape of the reconsolidation window itself.

## The Six Vehicles, in Causal Order

### 1. The Declaration Itself — What Is Being Returned To

**File:** [[Covenant Identity — Matched Declaration Architecture]] *(06 — Practitioner Reference)*

Before any return can happen, there has to be a specific thing to return to — the exact covenant truth text matched to the client's wound type and named false covenant object, not a generic verse. Every vehicle downstream in this sequence assumes this content already exists and is precisely worded. If this step is vague or generic, every later vehicle inherits that vagueness — receptive return to an imprecise declaration produces imprecise reorganization, per Condition 1 in the Reception mechanism.

**What breaks if missing:** The client has "something to think about" rather than a specific claim to bring before God. Every downstream vehicle is doing receptive work on a target that isn't sharp enough to displace anything.

---

### 2. The Immediate Window — Protecting What Just Opened

**File:** [[Covenant Identity — Post-Session Integration Guide]] *(08 — Between-Session Materials)*

The 24-48 hours immediately following the session are the most neurologically active window — the implicit encoding is still labile from the Node 5-6 event. This vehicle governs that window specifically: body care first, then receptive prayer, then a Scripture anchor, then brief naming, then one safe person — in that order, because a Warrior-type client will skip straight to cognitive analysis, a Hermit will skip to isolation, and a False Noble will skip to premature resolution, each of which closes the window before it's had its full effect.

**What breaks if missing:** The client re-armors, shuts down, or manages the encounter away within hours of it happening, and the window that was still open closes on its own default pattern rather than being protected.

---

### 3. The Daily Structural Practice — Where Receptive Return Actually Happens

**File:** [[Covenant Identity — Spiritual Disciplines Guide]] *(08 — Between-Session Materials)*
**Wording for the instruction itself:** [[Covenant Identity — Assigning Receptive Return — Practitioner Technique]]

This is the vehicle that carries the actual daily receptive-return practice described in the Consolidation mechanism — the disciplines (solitude, silence, prayer) are the structural container in which the client brings the declaration before the Father, speaks it once, stops, and waits. This is also where the wound-type co-optation risk is highest and most consequential: a Warrior can turn the discipline into a new competence project, a Hermit can use solitude to reinforce isolation rather than encounter, a False Noble can turn stillness into a new performance standard. The Assigning Receptive Return document's calibration table exists specifically to guard this vehicle.

**What breaks if missing:** There is no actual daily vehicle for receptive return to occur in — the client has a declaration and a protected first 48 hours, but nothing carrying the practice forward across the weeks between sessions, which is where the Consolidation mechanism says the real reorganization accumulates.

---

### 4. Ongoing Attentiveness — Feeding the Sequence Back

**File:** [[Covenant Identity — Phase-Specific Reflection Guide]] *(08 — Between-Session Materials)*

This vehicle is not where receptive return happens — it's where the client (and, through session report, the practitioner) notices whether it's happening and what it's producing. Noticing, not processing: "what is present" rather than "what does this mean." This is also the lowest-Spirit-dependency, most easily secularized vehicle in the sequence, which is why the Spirit-invitation opening line is named as non-optional rather than a nice-to-have.

**What breaks if missing:** Neither the client nor the practitioner has a reliable channel for detecting whether vehicle 3 is producing receptive return or affirmation repetition until the next session's verbal report — which is a much slower and less specific feedback loop.

---

### 5. The Relational Channel — Reaching What Private Practice Cannot

**File:** [[Covenant Identity — Accountability Companion Guide]] *(08 — Between-Session Materials)*

Because the original false covenant object was formed relationally (a wound, a betrayal, a conditional-reception pattern), some of the implicit reorganization is reached more effectively through a relational disconfirmation than a private one. This vehicle is not required for every client in the same way — the mechanism doc names it as most load-bearing specifically for the Hermit wound, where private solitude can reinforce rather than disconfirm the "proximity is threat" pattern — but it's available as a channel regardless of type.

**What breaks if missing:** For wound types where the false structure is organized around avoiding relational risk specifically, the entire consolidation sequence stays private, which can leave the exact pattern that needs relational disconfirmation untouched by an otherwise well-run private practice.

---

### 6. The Theological Frame Underneath All Five

**File:** [[Covenant Identity — John 15 Abide Formation Posture Reference]] *(06 — Practitioner Reference)*

This isn't a vehicle in the same sense as 1-5 — it's the frame that keeps all of them from collapsing into striving. It's why none of the above are assigned as homework to complete but as a posture of *remaining* to keep re-entering. A practitioner who hasn't oriented a client (and themselves) to this frame risks the client executing every vehicle above as production rather than reception — which is precisely the affirmation-repetition failure mode the whole sequence exists to avoid.

**What breaks if missing:** The client can execute vehicles 1-5 with technical correctness while still operating in an achievement posture toward all of them — doing receptive return "right" as a new project, which the Warrior and False Noble calibration notes in vehicle 3 already flag as the most common failure mode this frame exists to prevent.

## Diagnostic Use — When Consolidation Has Stalled

When a client reports faithful between-session engagement with no felt change, check the sequence in order rather than assuming the whole mechanism has failed:

| Check | Question | If missing |
|---|---|---|
| 1 — Declaration | Is the content specific enough (Condition 1), or still generic? | Return to Stage 1-2 work before reassigning anything downstream |
| 2 — Immediate window | Was the 24-48 hour window protected, or did the client re-armor/withdraw/resolve immediately? | Re-issue Post-Session Integration Guide explicitly next time |
| 3 — Daily practice | Is this receptive return or affirmation repetition? (See Assigning Receptive Return technique doc's diagnostic question.) | Re-word the instruction; check for wound-type co-optation |
| 4 — Attentiveness | Does the client have any channel for noticing what's happening, or only the practice itself? | Assign or re-emphasize Phase-Specific Reflection Guide |
| 5 — Relational | Is this a wound type (especially Hermit) where private practice alone cannot reach the pattern? | Introduce or revisit the Accountability Companion |
| 6 — Frame | Is the client executing this as production or reception? | Return to the John 15 frame explicitly before continuing any vehicle |
