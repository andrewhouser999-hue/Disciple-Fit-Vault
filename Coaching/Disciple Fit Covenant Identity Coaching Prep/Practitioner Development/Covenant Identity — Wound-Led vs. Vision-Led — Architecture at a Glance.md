# Covenant Identity — Wound-Led vs. Vision-Led — Architecture at a Glance

*What it is: a one-page orientation to how the CIC diagnostic sequence handles a client working from pain versus a client working from purely growth. Companion to [[Covenant Identity — Diagnostic Lens Transition Logic]], which holds the full mechanism, design rationale, and contested ground — this document is the fast-reference summary, not a substitute for it.*

*When used: re-orient quickly before a Discovery Call with an ambiguous presentation, or as a refresher on the architecture without re-reading the full addendum.*

---

## The governing design

One diagnostic engine, two front doors. Both routes are aimed at the same target — a specifically named functional dependency (false covenant object) — met by the same covenant truth. The difference is concentrated at the entry point and one activation condition; everything downstream is identical.

---

## Where they differ

| | **Wound-Led (pain)** | **Vision-Led (purely growth)** |
|---|---|---|
| **Entry read** | Client names, or the discovery-call/intake surfaces, a presenting cost in life, identity, relational, or internal domains | No presenting cost named; genuine desire to grow deeper, willing to be diagnosed |
| **Stage 1 mechanism** | Affections Discovery — tracks where cost is landing (what's failing under pressure) | Appetite Discovery — tracks where a *healthy-looking* appetite is actually routing (low-stakes Pursuit Question, Hiddenness Test, ITC improvement-goal opening) |
| **What's being surfaced** | A specific, named functional dependency (false covenant object) | Same target — only the surfacing route differs |
| **Theological framing** | Rescue from a structure that's failing | Invitation into more than what's currently being lived — telos isn't problem-resolution, so a non-crisis client is inside scope, not an edge case |
| **Stage 2→3 activation (Condition 2)** | Grief — weariness, fatigue directed at the false structure itself | Vision Gap Awareness (Condition 2-Alt) — a *felt* gap between who the client wants to become and who they are now; longing, not grief. Same rigor, not a relaxed substitute |
| **Unique safeguard** | None needed — the Wound-Led picture is self-evident | Lateral Reroute Signal — checks whether the "growth-seeking" is itself a false covenant object (compulsive, KPI-tracked, performs for an audience). If so, reroute back to Wound-Led |
| **Intake signal** | Q1/Q2/Q6 read normally; Go/No-Go Criterion 1 reads as Contemplation | Q1/Q6 carry an added clause inviting the no-problem answer; Criterion 1 has its own Vision-Led branch so the client isn't wrongly scored Pre-Contemplation and screened out |

---

## Where they don't differ

- **Stage 2 (False Identity Named), Stage 3 (Covenant Truth), Stage 4a/4b, and the Referral Signal** — identical machinery regardless of which door the client entered through.
- **The four-phase engagement container** (Covenant Orientation → Wound Mapping → Identity Installation → Integration & Close) — phase names stay as internal/practitioner labels; a Vision-Led client is never told they're in "Wound Mapping," same as they'd never hear "formation apprentice."
- **The diagnostic target itself** — a specifically named false covenant object. Neither track produces a different *kind* of outcome; they produce the same outcome by different roads.

---

## A third axis, orthogonal to the front door (added 2026-07-14)

The table above describes the Stage 1 entry fork — which door the client walks through. There is a second, independent fork that was previously (and incorrectly) treated as automatic: **whether Character Wound Deepening applies at all.**

This axis cuts across both doors, not along the Wound-Led/Vision-Led line:

| | Injury-shaped wound present | No injury-shaped wound |
|---|---|---|
| **Wound-Led entry** | Character Wound Diagnostic Tool applies as originally designed — Warrior/Hermit/False Noble | Skip the tool; name the false identity directly from the affections picture (this is common — see [[Sunday-Monday Christian]]) |
| **Vision-Led entry** | Rare but possible — the tool still applies if a specific injury surfaces during appetite-tracking | The typical case; skip the tool, name directly from the appetite picture |

Previously, [[Covenant Identity — Diagnostic Lens Transition Logic]] stated Character Wound Deepening "applies the same way" once the Vision-Led picture is specific enough — implying it was automatic for both doors. That was never checked against the tool itself, which is built entirely from three injury archetypes (betrayal, terror, ambivalence). It has been corrected: deepening is now conditional on an actual injury signal, checked via [[Covenant Identity — Character Wound Diagnostic Tool]]'s "Before Typing" section, regardless of which door the client entered through.

---

## The one-sentence version

Pain-driven clients get found through what's costing them; growth-driven clients get found through where a functioning appetite is quietly misrouted — but both are heading toward the same named false identity, met by the same covenant truth, activated by the same requirement (the false structure must be *felt* as insufficient, not just known to be) — grief supplies that felt insufficiency for one population, holy discontent supplies it for the other.
