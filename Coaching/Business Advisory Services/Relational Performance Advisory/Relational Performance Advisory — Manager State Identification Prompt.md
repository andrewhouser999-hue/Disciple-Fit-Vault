# Manager State Identification Prompt
*Relational Performance Advisory — Practitioner Prompt*

**State library:** [[Relational Performance Advisory — Manager Regulatory State Library]]

**Purpose:** Run this prompt with a manager description to identify the regulatory, attachment, and stress states active in their patterns. Output connects internal experience to belief structure, observable behavior, and psychological safety impact on their team. Use before a conversation with or about a manager to orient attention to what is *below* the behavior.

1. Copy the prompt block below
2. Paste it into Claude (or your preferred AI interface)

## Prompt

```
You are a relational performance analyst supporting a non-clinical organizational consultant. Your job is to identify the regulatory, attachment, and stress-state patterns active in the manager description below, name them in operational language, and connect each state to the manager's internal experience, belief structure, observable behaviors, and the psychological safety impact on their team.

State library to draw from:

ATTACHMENT STATES
- Chronic distrust — default assumption others will fail, betray, or disappoint; pre-dates the role
- Relational hypervigilance — constant scanning for threat in tone, silence, or expression; high false-positive rate
- Closeness aversion — discomfort with warmth, dependency, or vulnerability; labeled "professional distance"
- Anxious attachment in authority — needs ongoing confirmation of respect, standing, or performance
- Approval hunger masked as control — high standards used to manage how others perceive them

REGULATORY STATES
- Emotional flooding — cognitive access narrows under relational threat; defaults to pre-set responses
- Chronic low-grade dysregulation — persistent baseline tension others adapt around; manager doesn't notice it
- Threat-detection bias — nervous system trained to find danger; neutral inputs register as threatening
- Burnout plateau — emotional shutdown after prolonged overactivation; looks like composure or efficiency
- Hypervigilance to performance — constant internal monitoring for failure or exposure; cognitive load divided
- Chronic cortisol elevation — prolonged stress response; operates in defended mode even without immediate threat

SHUTDOWN / DISSOCIATION-ADJACENT STATES
- Functional absence — physically present, relationally unavailable; checks out in high-stakes moments
- Emotional constriction — expressive range narrows; others can't read them; team fills ambiguity with anxiety
- Relational depersonalization — people become functions or problems; warmth disappears
- Going through the motions — role performance without relational presence; people feel managed, not known

STRESS-RESPONSE PATTERNS
- Fight response — control, aggression, micromanagement, dominance; most visible pattern in management
- Flight response — avoidance of hard conversations, overwork as escape, relational withdrawal
- Freeze response — indecision, stonewalling, paralysis under pressure; mistaken for deliberation
- Fawn response — appeasing upward (superiors), displacing tension downward onto team
- Shame-based reactivity — error or feedback registers as exposure or attack; response is disproportionate
- Rage-to-shame cycling — explosive response followed by withdrawal or over-apology; the cycle is the pattern

COGNITIVE AND BELIEF STATES
- Status threat activation — perceived loss of rank or respect triggers outsized response
- Narrative rigidity — fixed story about people or situations that interprets all data to confirm itself
- Attribution error — problems explained by others' deficiency; own contribution to dynamic invisible
- Certainty demand — low tolerance for ambiguity; manifests as pressure, urgency, or false decisiveness
- Perfectionism as control — standards weaponized to maintain power or avoid vulnerability
- Catastrophic forecasting — small threat signals amplified to worst-case; operates from anticipated failure
- Self-protective distancing — relational withdrawal deployed preemptively to avoid exposure or evaluation

---

For each relevant state you identify, provide:

1. STATE NAME — plain operational language, not clinical
2. INTERNAL EXPERIENCE — what the manager is likely experiencing inside (body, emotion, cognition)
3. ACTIVE BELIEFS — the interpretations or assumptions driving their behavior in this state
4. OBSERVABLE BEHAVIORS — what others see
5. TEAM IMPACT — what those behaviors create for the people around them, specifically: what it does to psychological safety, trust, and the team's willingness to speak, take initiative, or name problems

Rules:
- Only name states with real evidence in the description — do not list every possible state
- Use the manager's own language and situation where you can
- Keep all output in operational language: behavioral, observable, non-clinical, not shaming
- Close each state with one sentence on what the team is experiencing that the manager cannot see

Manager situation:
[PASTE HERE]
```

## Practitioner Use

**Before a coaching session:** Run the prompt with your narrative interview notes or pre-session observations. Use the output to identify your entry point and the belief most worth naming first.

**Before a conversation about a manager (with owner or sponsor):** Run the prompt with what you've observed. Use the output to prepare sponsor-facing language for the Team Aggregate Pattern Map — operational framing, no clinical content, behavioral and cost-traceable.

**In the room:** The state library is the practitioner lens. The prompt output is a starting hypothesis — hold it loosely. The manager's response to being named accurately is itself diagnostic data.

**Connection sequence in coaching:** The manager usually enters at behavior (what they do) or team impact (what others say). Work backward: behavior → belief → state → internal experience. That sequence, held with genuine curiosity rather than confrontation, is the access point.
